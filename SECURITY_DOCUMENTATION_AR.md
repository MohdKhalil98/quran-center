# Firebase Security Rules - شرح وتطبيق

## نقاط الأمان الرئيسية:

### 1. Authentication (التحقق من الهوية)
✅ جميع العمليات تتطلب مستخدم مسجل دخول
✅ المستخدم لا يمكنه الوصول إلا لبياناته أو بيانات مركزه

### 2. Role-Based Access Control (RBAC)
- **Admin**: الوصول الكامل لكل البيانات
- **Supervisor**: الوصول لبيانات مركزه فقط
- **Teacher**: الوصول لبيانات طلابهم وحلقاتهم
- **Student**: الوصول لبياناتهم الشخصية فقط
- **Parent**: الوصول لبيانات أطفالهم (إذا كان مفعل)

### 3. Center Isolation (عزل المراكز)
كل موظف يرى فقط بيانات مركزه:
```javascript
resource.data.centerId == userCenterId()
```

### 4. User Data Privacy
- المستخدم يستطيع تعديل بيانات شخصية فقط (الاسم، الهاتف)
- لا يمكن تعديل الأدوار أو المعرّفات
- المشرفين فقط يستطيعون تعديل الأدوار

### 5. Messages Security
- كل شخص يرى رسائله فقط (مرسل ومستقبل)
- لا يسمح بحذف أو تعديل الرسائل المرسلة

## خطوات التطبيق:

### الخطوة 1: نسخ القواعد
1. اذهب إلى [Firebase Console](https://console.firebase.google.com)
2. اختر project "halqatmoza"
3. Firestore Database → Rules
4. انقر على "Edit rules"
5. احذف القواعس الحالية
6. الصق محتوى الملف `FIREBASE_SECURITY_RULES.txt`

### الخطوة 2: اختبر القواعس (Simulator)
قبل النشر:
1. افتح "Simulate" في Firebase Rules Editor
2. اختبر السيناريوهات التالية:
   - **Admin يقرأ users**: ✅ يجب أن ينجح
   - **Teacher يقرأ users خارج مركزه**: ❌ يجب أن يفشل
   - **Student يقرأ messageId غير مرسل له**: ❌ يجب أن يفشل
   - **Student يقرأ attendance خاصه**: ✅ يجب أن ينجح

### الخطوة 3: انشر القواعس
1. بعد التحقق، اضغط "Publish"
2. الآن القواعس تعمل على الإنتاج

## ملاحظات مهمة:

### ⚠️ عند إضافة collections جديدة:
أضف قاعدة لهذه collections أو سيكون الوصول الافتراضي "Allow Admin Only"

### ⚠️ عند تغيير البيانات:
إذا أضفت حقل جديد مثل `centerId`، تأكد أنه موجود في كل documents

### ⚠️ Performance:
هذه القواعس تستخدم `get()` وقد تؤثر على الأداء مع ملايين المستندات:
- استخدم Firestore Indexes إذا لزم الأمر
- Firebase سيخبرك بالحاجة للـ Indexes تلقائياً

### ✅ التحسينات المستقبلية:
1. أضف 2FA للمديرين
2. استخدم custom claims في Firebase Auth للأدوار
3. أضف audit logging (تسجيل جميع العمليات)
4. استخدم Firestore backup للبيانات المهمة

---

## الخلاصة:

✅ البيانات محمية بناءً على الأدوار والمركز
✅ المستخدمون يرون فقط بياناتهم المصرح لهم
✅ المشرفين والمعلمين لا يمكنهم الوصول لبيانات مراكز أخرى
✅ الرسائل محمية بين المرسل والمستقبل فقط
✅ لا حذف أو تعديل للرسائل المرسلة
